# Collaboration Playbook

이 문서는 팀원이 동일한 방식으로 개발/리뷰/설명을 할 수 있도록 만든 운영 가이드입니다.

## 설명 프레임 (다른 사람에게 공유할 때)
1. 문제 정의: 어떤 사용자 문제를 해결하는가?
2. 접근 방식: 왜 이 구조(VQA + Bot Detection 분리)인가?
3. 검증 방법: 어떤 지표로 성공/실패를 판단하는가?
4. 리스크: 오탐/미탐, 데이터 편향, 재현성 문제를 어떻게 관리하는가?

## PR 발표 템플릿
- 목적: 이 PR이 해결하는 단일 문제
- 변경: 핵심 파일 3개 이내 설명
- 검증: 테스트/지표/재현 절차
- 리스크: 회귀 가능성과 롤백 방법

## 리뷰 관점
- 정확성: 요구사항을 올바르게 만족하는가?
- 안정성: 회귀 가능성이 낮은가?
- 재현성: 동일 조건에서 같은 결과가 나오는가?
- 운영성: 추적 가능한 로그/메트릭이 있는가?
